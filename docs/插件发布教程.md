# FlowMD 插件发布教程

这份文档说明如何把当前 FlowMD 扩展发布到 VS Code Marketplace。  
如果你只是想本地安装测试，也可以只执行“打包 VSIX”那一步，不必走到最终发布。

## 先准备什么

发布一个 VS Code 扩展，至少要准备下面这些东西：

1. 一个 Microsoft 账号，用来登录 Marketplace 和创建发布者。
2. 一个 Azure DevOps / Marketplace 发布用的 Personal Access Token，简称 `PAT`。
3. 一个 Marketplace Publisher，也就是发布者身份。
4. 本机的 Node.js 和 npm。
5. `vsce` 工具，用来打包和发布扩展。
6. 扩展仓库里的基础发布资源：
   - `package.json`
   - `README.md`
   - 图标文件 `icon.png`
   - `LICENSE`
   - 可选的 `CHANGELOG.md`

## 官方网址

1. VS Code 发布扩展官方文档

## 第一步：准备本机环境

先确认本机有 Node.js 和 npm。

检查命令：

```powershell
node -v
npm -v
```

如果命令能输出版本号，说明环境基本正常。

## 第二步：准备扩展基础信息

在这个仓库里，发布信息主要来自 `package.json`。

你需要重点确认这些字段：

1. `name`
2. `displayName`
3. `description`
4. `version`
5. `publisher`
6. `main`
7. `activationEvents`
8. `categories`
9. `icon`
10. `repository`
11. `bugs`
12. `homepage`

### 这些字段为什么重要

1. `publisher` 必须和 Marketplace 上创建的发布者 ID 一致。
2. `version` 每次发布都要递增。
3. `icon` 最好准备一个 128x128 以上的 PNG 图标。
4. `README.md` 会显示在 Marketplace 页面。
5. `LICENSE` 是发布合规性常见要求。

## 第三步：创建 Personal Access Token

`PAT` 是发布时最关键的登录凭据。

### 创建入口

1. 打开 Azure DevOps 个人设置。
2. 进入 `Personal access tokens`。
3. 点击 `New Token`。

### 推荐设置

根据 VS Code 官方文档，推荐这样填：

1. `Name`：随便起一个容易识别的名字。
2. `Organization`：选择 `All accessible organizations`。
3. `Expiration`：建议设置一个你能接受的有效期。
4. `Scopes`：展开 `Show all scopes`，找到 `Marketplace`，勾选 `Manage`。

### 官方参考

1. Azure DevOps PAT 文档  
   [Use personal access tokens](https://learn.microsoft.com/en-us/azure/devops/organizations/accounts/use-personal-access-tokens-to-authenticate?view=azure-devops)
2. VS Code 发布文档中的 PAT 说明  
   [Publishing Extensions](https://code.visualstudio.com/api/working-with-extensions/publishing-extension)

### 你要记住的点

1. 这个 token 只在创建时可见一次。
2. 一定要保存好。
3. 发布失败时，最常见原因就是 `Organization` 选错，或者 `Marketplace (Manage)` 权限没勾对。

## 第四步：创建 Publisher

Publisher 是你在 Marketplace 上的“发布者身份”。

### 创建入口

1. 打开 [Marketplace 管理页](https://marketplace.visualstudio.com/manage)。
2. 用创建 `PAT` 时使用的同一个 Microsoft 账号登录。
3. 点击 `Create publisher`。
4. 填写发布者信息。

### 推荐填写方式

1. `ID`：建议和仓库里的 `publisher` 保持一致。
2. `Name`：显示名称，可以是你的品牌名或个人名。

### 本项目对应关系

当前这个仓库里的 `package.json` 里已经写了：

```json
"publisher": "lyjy-kq"
```

所以 Marketplace 上的 Publisher ID 也建议创建成 `lyjy-kq`，这样最省事。

### 官方参考

1. [Publishing Extensions](https://code.visualstudio.com/api/working-with-extensions/publishing-extension)

## 第五步：检查发布资源

发布前，建议确认下面这些文件都在仓库根目录，或者至少和发布清单里写的一致：

1. `package.json`
2. `README.md`
3. `LICENSE`
4. 图标文件
5. `dist/extension.js`
6. `dist/webview.js`

### 重点说明

1. `README.md` 会作为 Marketplace 页面主体说明。
2. `icon` 最好是 PNG。
3. 如果你改过仓库结构，要同步检查 `extension.vsixmanifest` 和 `package.json` 里的路径。

## 第六步：先打包 VSIX

如果你还没准备好直接发布，可以先打包一个 `.vsix` 文件，本地安装测试。

本仓库可以直接用下面命令打包：

```powershell
npx @vscode/vsce package --no-dependencies --skip-license -o .\flow-md.vsix
```

打包成功后，你会得到一个 `flow-md.vsix` 文件。

### 这一步的用途

1. 本地安装测试。
2. 先确认包内容是否正确。
3. 也可以作为手动上传 Marketplace 的包。

## 第七步：两种发布方式

### 方式 A：直接发布

适合你已经准备好 `PAT`，并且想一步到位发布。

```powershell
$env:VSCE_PAT="你的PAT令牌"
npx @vscode/vsce publish --no-dependencies --skip-license
```

如果你已经登录过，也可以先执行：

```powershell
npx @vscode/vsce login lyjy-kq
```

然后再执行：

```powershell
npx @vscode/vsce publish --no-dependencies --skip-license
```

### 方式 B：先打包，再手动上传

适合你想先看成品，再上传到 Marketplace。

1. 先打包：

```powershell
npx @vscode/vsce package --no-dependencies --skip-license -o .\flow-md.vsix
```

2. 打开 [Marketplace 管理页](https://marketplace.visualstudio.com/manage)。
3. 找到你的 Publisher。
4. 上传 `flow-md.vsix`。

## 第八步：版本号怎么处理

每次发布前，建议把 `package.json` 里的 `version` 提高一个版本号。

例如：

- `0.3.11` -> `0.3.12`
- `0.3.11` -> `0.4.0`
- `0.3.11` -> `1.0.0`

如果你用 `vsce publish`，它也支持自动递增版本号。

## 常见问题

### 1. 提示 401 / 403

通常是这几个原因：

1. `PAT` 过期了。
2. `Organization` 没选成 `All accessible organizations`。
3. `Scopes` 没勾 `Marketplace (Manage)`。
4. 登录的 Microsoft 账号和创建 Publisher 的账号不是同一个。

### 2. 提示 extension name 已存在

说明 Marketplace 里已经有同名扩展了，需要改 `package.json` 里的 `name`，或者确认是不是你自己的旧版本。

### 3. 打包时找不到图标或 README

检查 `package.json` 和 `extension.vsixmanifest` 里的路径是否和实际文件一致。

### 4. `vsce` 报依赖问题

先确认仓库能正常执行构建：

```powershell
npm run build
```

如果构建没问题，再继续打包。

## 推荐的最短流程

如果你想最快完成一次发布，可以按这个顺序走：

1. 准备 Microsoft 账号。
2. 创建 `PAT`，scope 选 `Marketplace (Manage)`。
3. 去 [Marketplace 管理页](https://marketplace.visualstudio.com/manage) 创建 Publisher。
4. 确认 `package.json` 里的 `publisher` 一致。
5. 执行 `npx @vscode/vsce package --no-dependencies --skip-license -o .\flow-md.vsix`。
6. 本地测试没问题后，执行 `npx @vscode/vsce publish --no-dependencies --skip-license`。

## 你现在这个仓库的建议

当前仓库已经能打出 `.vsix` 包了，所以你下一步最现实的操作是：

1. 先在 VS Code 里本地安装这个 `.vsix` 测试。
2. 如果没问题，再去准备 `PAT`。
3. 最后执行发布。
